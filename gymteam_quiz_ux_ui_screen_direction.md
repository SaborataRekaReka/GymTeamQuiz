# UX/UI режиссура экранов квиз-воронки GymTeam / Екатерина Усманова

Документ задаёт визуальную и UX-подачу экранов для AI-агента и разработчика. Он не заменяет исходное ТЗ. Все пользовательские тексты, вопросы, подзаголовки, варианты ответов, тарифы, юридические ссылки и порядок экранов брать только из исходного ТЗ и `data/quiz_screen_map.json`.

Главный принцип: делаем аккуратный мобильный квиз в духе Harna Walking Program по UX-логике, но без копирования чужого бренда, текстов, изображений и фирменных элементов.

## 1. Жёсткие правила соответствия ТЗ

1. Порядок экранов не менять.
2. Смысл вопросов и варианты ответов не менять.
3. Пользовательские тексты из ТЗ вставлять как есть.
4. Новые пользовательские заголовки, подзаголовки, продающие фразы, предупреждения, бейджи и подписи не добавлять без отдельного согласования.
5. Служебные элементы интерфейса допустимы только там, где без них нельзя пройти квиз: кнопка продолжения на множественном выборе, кнопка отправки формы, кнопка показа результата, кнопка оплаты или перехода к оплате.
6. В пользовательском интерфейсе нельзя использовать английские слова.
7. Кавычки только «ёлочки».
8. Не использовать длинные тире как разделители внутри предложения.
9. Не использовать стрелки в пользовательских текстах.
10. Обращение только на «вы».
11. Не обесценивать возраст, вес и тело.
12. Тёмная тема запрещена. Фон светлый.
13. Не больше двух акцентных цветов.
14. Без мигающих и прыгающих элементов.
15. Без надписей «осталось N мест».
16. Без регистрации до результата. Почта запрашивается один раз на лид-гейте.
17. Чекбокс согласия ставится только на лид-гейте.
18. Форма оплаты своя не верстается. На пейволле должен быть блок для эмбеда формы ГетКурса.
19. Прогресс показывается процентом или сегментами, без фразы «вопрос N из N».
20. Все изображения должны быть сжаты и не должны тормозить прохождение.

Технические слова из этого документа вроде «экран», «карточка», «состояние выбора», «нижняя кнопка», «прогресс», «бейдж раздела», «мобильная ширина» являются внутренними терминами. Их нельзя автоматически выводить пользователю, если они не указаны в ТЗ.

## 2. Общий визуальный язык

Настроение: светлый, чистый, женственный, уверенный, заботливый фитнес-продукт. Интерфейс должен ощущаться как персональный подбор, а не как сухая анкета.

Референс по UX-приёмам: Harna Walking Program.

Берём из референса:

- центрированный мобильный контейнер;
- много воздуха;
- крупный заголовок;
- короткий поясняющий текст;
- сегментированный прогресс;
- круглая кнопка назад;
- крупные карточки ответов;
- заметное выбранное состояние;
- чекбоксы справа в множественном выборе;
- крупный числовой ввод по центру;
- объясняющие карточки под числовым вводом;
- переходные экраны с крупным изображением, заголовком, текстом и кнопкой;
- нижнюю фиксированную кнопку на длинных экранах.

Не берём из референса:

- английские пользовательские слова;
- бренд Harna;
- чужие изображения;
- чужие иконки без проверки прав;
- агрессивные медицинские или телесные визуалы;
- сложную анимацию;
- перегруженный paywall.

## 3. Базовая сетка и компоненты

### Контейнер

- Максимальная ширина контента: 560–640 px на десктопе.
- На мобильном: ширина 100%, внутренние отступы 20–24 px.
- Основной контент центрируется.
- Длинные экраны прокручиваются, нижняя кнопка остаётся доступной.

### Верхняя зона

Использовать единый верхний блок:

- маленький бейдж раздела по смыслу экрана;
- кнопка назад в круге слева, кроме первого экрана;
- прогресс: сегментированная линия или процентная линия;
- без номера вопроса.

Бейдж раздела является служебным UI-элементом. Он не должен добавлять новый маркетинговый смысл. Допустимые внутренние названия разделов: «СТАРТ», «ТЕЛО», «ЦЕЛЬ», «ПРИВЫЧКИ», «ПРОФИЛЬ», «ПЛАН», «ДОСТУП». Если есть риск, что это будет считаться новым текстом, бейдж можно скрыть.

### Заголовок и подзаголовок

- Заголовок: 28–32 px на мобильном, жирный, цвет тёмный.
- Подзаголовок: 15–17 px, цвет мягкий серый.
- Заголовки и подзаголовки брать из ТЗ.
- Не переписывать тексты для красоты.
- Не добавлять новые эмоциональные фразы.

### Карточки ответов

Обычное состояние:

- фон белый;
- бордер светло-серый;
- радиус 22–28 px;
- высота 68–88 px;
- текст 16–18 px;
- текст полужирный;
- иконка или мини-изображение слева допустимы, если не меняют смысл.

Выбранное состояние:

- бордер 2 px тёмно-серый или акцентный;
- фон очень светло-серый или очень мягкий оттенок акцента;
- чекбокс или галочка справа для множественного выбора;
- выбранная карточка должна быть заметна сразу.

### Кнопки

- Главная кнопка: широкая, высота 56–64 px, радиус 28–32 px.
- Цвет: насыщенный синий или другой основной акцент, выбранный в теме.
- На длинных экранах кнопка фиксируется внизу с белой подложкой.
- На одиночном выборе можно переходить по тапу карточки без отдельной кнопки.
- На множественном выборе и числовом вводе кнопка обязательна.
- Если текст кнопки не задан в ТЗ, использовать нейтральный служебный текст и вынести его в конфиг для согласования.

### Числовой ввод

- Крупное значение по центру.
- Единица измерения рядом: кг или см.
- Под значением тонкая линия.
- Под вводом поясняющая карточка, текст только из ТЗ или минимальный служебный текст, согласованный отдельно.
- Клавиатура numeric на мобильном.
- Мягкая валидация диапазона без пугающих сообщений.

### Переходные экраны

- Крупное изображение сверху.
- Заголовок из ТЗ.
- Текст из ТЗ.
- Кнопка «Продолжить», если она указана в ТЗ.
- Много воздуха.
- Без дополнительных продающих блоков.

### Лоадеры

- Длительность 2,5–4 секунды.
- Показывать подпункты из ТЗ.
- Подпункты подсвечиваются по очереди.
- После завершения автоматический переход дальше.
- Анимация плавная и лёгкая.

## 4. Режиссура экранов

## Экран 1. Старт, возраст

Источник текста: исходное ТЗ.

Формат: одиночный выбор.

Визуальная задача: мягкий вход в персональный подбор. Экран должен сразу показать, что это программа Кати Усмановой, а не обычная анкета.

Композиция:

1. Верхняя зона с прогрессом.
2. Оффер над вопросом из ТЗ.
3. Фото Кати Усмановой и логотип «Усманова Фит».
4. Вопрос из ТЗ.
5. Подзаголовок из ТЗ.
6. Карточки возрастных диапазонов.
7. Мелкая строка согласия со ссылками внизу, без чекбокса.

UX:

- Тап по варианту выбирает возраст.
- После выбора карточка подсвечивается.
- Переход дальше можно делать после короткой задержки.
- Если используется кнопка, она появляется после выбора.

Визуал:

- Фото Кати в верхнем блоке, аккуратная скруглённая карточка.
- Карточки возраста без тяжёлых картинок.
- Логотип не должен спорить с вопросом.

Запрещено:

- чекбокс согласия на этом экране;
- новые обещания, которых нет в ТЗ;
- агрессивные формулировки про возраст.

## Экран 2. Фигура сейчас

Формат: одиночный выбор.

Визуальная задача: дать человеку выбрать текущую точку без стыда и оценки.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки вариантов.

UX:

- Тап по карточке выбирает вариант.
- Карточка подсвечивается.
- Переход дальше после выбора.

Визуал:

- Можно использовать нейтральные мягкие иллюстрации силуэта слева.
- Если иллюстрации не готовы, лучше сделать текстовые карточки с небольшими иконками.
- Иллюстрации не должны выглядеть как оценка тела.

Запрещено:

- фото проблемных зон крупным планом;
- сравнения «плохо/хорошо»;
- визуальное унижение текущей формы.

## Экран 3. Цель по телу

Формат: одиночный выбор.

Визуальная задача: перевести внимание с проблемы на желаемый результат.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки целей.

UX:

- Одиночный выбор.
- Выбор фиксируется в состоянии квиза как ключевая цель.
- Переход дальше после выбора.

Визуал:

- Каждая карточка может иметь маленькую иконку по смыслу: лёгкость, тонус, формы, здоровье спины.
- Иконки должны быть одинакового стиля.
- Выбранное состояние заметное.

Запрещено:

- добавлять свои названия целей;
- менять порядок вариантов;
- усиливать обещание результата сверх ТЗ.

## Экран 4. Зоны

Формат: множественный выбор.

Визуальная задача: дать выбрать фокус тренировки. Экран может быть похож на Harna-экран с зонами тела, но с более бережной подачей.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Список карточек зон.
5. Нижняя кнопка продолжения.

UX:

- Можно выбрать несколько вариантов.
- Кнопка неактивна, пока ничего не выбрано.
- Если выбран вариант «Всё тело», логика должна быть определена в коде. Предпочтительно снимать остальные зоны, чтобы не было конфликта.
- Если пользователь после «Всё тело» выбирает конкретную зону, вариант «Всё тело» снимается.

Визуал:

- Карточки с чекбоксом справа.
- Можно добавить мягкие иконки зон слева.
- Не использовать жёсткие фотографии проблемных зон.
- На длинном списке кнопка фиксируется внизу.

Запрещено:

- добавлять новые зоны;
- использовать телесные изображения, которые вызывают стыд;
- делать кнопку недоступной визуально непонятно.

## Экран 5. Переходный экран доверия

Формат: экран доверия без выбора.

Визуальная задача: дать паузу и усилить доверие через число женщин, которые уже тренируются с Катей.

Композиция:

1. Крупное фото группы улыбающихся женщин на тренировке.
2. Заголовок из ТЗ.
3. Текст из ТЗ.
4. Кнопка «Продолжить».

UX:

- Никаких вариантов выбора.
- Кнопка ведёт на следующий экран.

Визуал:

- Фото в большой скруглённой карточке.
- Число 580 000 можно визуально выделить весом или цветом, но сам текст не менять.
- Экран должен быть спокойным и просторным.

Запрещено:

- добавлять новые цифры;
- добавлять неподтверждённые регалии;
- добавлять отзывы на этот экран, если они не предоставлены.

## Экран 6. Целлюлит

Формат: одиночный выбор.

Визуальная задача: деликатно собрать чувствительный ответ.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки вариантов.

UX:

- Одиночный выбор.
- Переход дальше после выбора.

Визуал:

- Лучше текстовые карточки с нейтральной иконкой.
- Можно использовать мягкий индикатор интенсивности в карточке, но без новых слов.
- Цвета спокойные.

Запрещено:

- фото целлюлита крупным планом;
- драматизация проблемы;
- медицинские обещания.

## Экран 7. Прошлая форма

Формат: одиночный выбор.

Визуальная задача: показать, что ответ нужен для настройки пути, а не для оценки прошлого.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Пять карточек вариантов.

UX:

- Одиночный выбор.
- Переход после выбора.

Визуал:

- Карточки можно оформить как временную линию: маленький маркер слева и текст справа.
- Вариант «До родов» оформлять нейтрально, без отдельного эмоционального давления.

Запрещено:

- добавлять комментарии про возраст;
- менять формулировку «До родов»;
- визуально выделять один вариант как более правильный.

## Экран 8. Активность

Формат: одиночный выбор.

Визуальная задача: собрать уровень активности для расчёта и подбора нагрузки.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки вариантов.

UX:

- Одиночный выбор.
- Ответ влияет на коэффициент активности.
- Переход после выбора.

Визуал:

- Слева можно использовать индикатор из 1–4 полосок.
- Полоски показывают уровень движения, без новых подписей.
- Стиль как на референсе Harna с activity-level, но текст только из ТЗ.

Запрещено:

- добавлять уровни «новичок/профи», которых нет в ТЗ;
- оценивать низкую активность негативно.

## Экран 9. Сон

Формат: одиночный выбор.

Визуальная задача: объяснить через подзаголовок из ТЗ, что сон учитывается в плане.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки вариантов.

UX:

- Одиночный выбор.
- Переход после выбора.

Визуал:

- Карточки простые, без изображений людей.
- Состояние выбора стандартное.

Запрещено:

- добавлять диагнозы;
- пугать последствиями недосыпа;
- менять тон на менторский.

Композиция:

- Кнопка ведёт дальше.
- Экран должен восприниматься как короткое объяснение.

Визуал:

- Фото Кати в скруглённой карточке.
- Можно использовать спокойную светлую подложку.
- Текст расположен под изображением.

Запрещено:

- добавлять неподтверждённые медицинские утверждения;
- добавлять новые преимущества;
- перегружать экран инфографикой.

## Экран 11. Результат-фокус

Формат: одиночный выбор.

Визуальная задача: уточнить, какой итоговый результат показывать в конце.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Четыре карточки вариантов.

UX:

- Одиночный выбор.
- Ответ влияет на визуализацию результата.
- Переход после выбора.

Визуал:

- Карточки с маленькими иконками результата.
- Не повторять визуально экран 3 полностью. Здесь акцент на итоговой визуализации, а не на образе тела.

Запрещено:

- смешивать этот ответ с целью по телу без явной логики в коде;
- менять формулировки вариантов.

## Экран 12. Текущий вес

Формат: числовой ввод.

Визуальная задача: сделать ввод веса простым и не тревожным.

Композиция:

1. Верхняя зона с прогрессом.
2. Вопрос из ТЗ.
3. Подзаголовок из ТЗ.
4. Крупный ввод числа.
5. Единица «кг».
6. Кнопка продолжения.

UX:

- Только числа.
- Диапазон по умолчанию: 35–250 кг.
- Если значение вне диапазона, показывать мягкую ошибку.
- Кнопка активна только после валидного значения.

Визуал:

- Центрированный крупный ввод как в референсе Harna.
- Тонкая линия под числом.
- Поясняющая карточка допустима только с текстом из ТЗ или согласованным служебным текстом.

Запрещено:

- шутки про вес;
- оценочные цвета;
- громкие предупреждения.

## 5. Сценарии взаимодействия

### Одиночный выбор

- Пользователь нажимает карточку.
- Карточка получает выбранное состояние.
- Ответ сохраняется.
- Переход дальше происходит автоматически после короткой задержки или через кнопку, если так реализована общая логика.

### Множественный выбор

- Пользователь выбирает одну или несколько карточек.
- Выбранные карточки получают галочку и усиленный бордер.
- Кнопка продолжения активна только при выборе минимум одного варианта.
- Повторный тап снимает выбор.

### Числовой ввод

- Пользователь вводит число.
- Единица измерения видна рядом с числом.
- Значение валидируется.
- Кнопка продолжения активна только при корректном значении.

### Лид-гейт

- Имя обязательно.
- Почта обязательна и валидна.
- Чекбокс согласия обязателен.
- Юридические ссылки открываются.
- После отправки сохраняется полный набор данных.

### Результат

- Все подстановки должны работать.
- Норма калорий и КБЖУ рассчитываются.
- ИМТ рассчитывается.
- Подборка программ соответствует ответам.

### Пейволл

- Все обязательные блоки присутствуют.
- Форма ГетКурса встроена или оставлено явное место под эмбед с технической пометкой.
- Юридические ссылки присутствуют.

## Общий принцип
