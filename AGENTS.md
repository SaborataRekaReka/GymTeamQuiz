# AGENTS.md

Ты AI-агент, который помогает разработчику собрать тестовое задание: мобильную квиз-воронку для подписки программ Екатерины Усмановой.

## Главная цель

Собрать рабочую воронку в Chatium:

1. Пользователь проходит квиз.
2. Ответы сохраняются.
3. Перед результатом пользователь вводит имя и почту.
4. Пользователь видит персональный результат.
5. Пользователь попадает на пейволл.
6. На пейволле встроена форма оформления заказа из ГетКурса.

## Рабочая среда

Финальная среда: Chatium.

Предпочтительный подход: TypeScript, Server Driven UI, внутренняя база Chatium, мобильная сборка.

Можно использовать внешний редактор кода, Codex, Cursor, Claude, ChatGPT или другие AI-инструменты. Итоговый код должен быть пригоден для переноса в Chatium.

## Приоритеты

1. Сквозной проход от первого экрана до пейволла.
2. Сохранение ответов и лидов.
3. Персонализация результата.
4. Пейволл с обязательными блоками.
5. Мобильная скорость и читаемость.
6. Аккуратный визуал.

## Запрещено

Не делай отдельный сайт вместо реализации в Chatium.

Не меняй смысл вопросов, варианты ответов, порядок экранов, обязательные блоки пейволла и правила пользовательского текста.

Не используй тёмную тему.

Не добавляй фразы про ограниченное количество мест.

Не добавляй мигающие, прыгающие и тяжёлые элементы.

Не показывай прогресс как «вопрос 3 из 24». Показывай проценты.

Не проси почту до лид-гейта.

Не пиши, что результат будет отправлен на почту. Почта нужна только для входа и сохранения лида.

Не размещай логотипы и ссылки на Instagram или Meta в интерфейсе квиза.

## Разрешено

Использовать настоящие фотографии Кати Усмановой с сайта проекта и её соцсетей, если они нужны как социальное доказательство.

Генерировать иллюстрации и вспомогательные изображения в AI-сервисах.

Делать композицию экранов самостоятельно.

Выбирать визуальные решения, если они не ломают ограничения ТЗ.

## Тон результата

Женская аудитория, цель: похудеть, убрать живот, вернуть форму.

Говорить на «вы».

Тон тёплый, поддерживающий, человеческий.

Фокус на желании пользователя и достижимом результате.

Без давления, стыда и менторства.

## Пользовательский текст

Пользовательский текст должен соблюдать правила из `docs/01_hard_constraints.md`.

Технические идентификаторы в коде могут быть на английском. Тексты, которые видит пользователь, должны быть на русском.
