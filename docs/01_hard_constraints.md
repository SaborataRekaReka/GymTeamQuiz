# 01. Жёсткие ограничения

## Среда разработки

Финальный квиз собирается внутри Chatium.

Chatium работает как облачная платформа, где приложения и страницы собираются на TypeScript по принципу Server Driven UI.

Не делать финальную версию отдельным сайтом на внешнем хостинге.

## Мобайл-первый

Около 90 процентов трафика будет с телефона.

Сначала довести мобильную версию. Десктоп проверять после мобильной.

Экран должен быть читаемым на телефоне, без мелких кнопок и перегруженных блоков.

## Скорость

Скорость загрузки критична.

Изображения должны быть сжаты.

Не использовать тяжёлые видео, большие гифки, мигающие элементы и сложные анимации.

## Лид-гейт

До результата регистрации нет.

Почта запрашивается один раз: только на экране 21.

На экране 21 нужен чекбокс согласия на обработку персональных данных.

Нельзя писать «пришлём на почту», потому что автоматическое письмо не отправляется.

## Прогресс

Прогресс показывать процентом.

Нельзя писать «вопрос такой-то из такого-то».

## Сохраняемые данные

Сохранять в базу: имя, e-mail, все ответы по экранам, рост, вес, целевой вес, присвоенный тип результата, посчитанную норму калорий, UTM-метки и дату.

## Пользовательский текст

Все видимые пользователю тексты должны соблюдать правила:

- кавычки только «ёлочки»;
- без обычных кавычек;
- без длинных тире как разделителей внутри предложения;
- без стрелок;
- вместо длинного тире использовать точку, двоеточие, «и», «а», «то есть»;
- без английских слов в текстах для людей;
- обращение на «вы»;
- без обесценивания возраста и тела;
- тон заботливый, без осуждения.

Технические имена переменных, ID экранов и комментарии в коде могут быть на английском.

## Визуал

Тёмные темы запрещены.

Фон светлый.

Не больше двух акцентных цветов.

Основной текст должен быть читаем на телефоне.

Кнопки должны быть заметными.

Запрещены мигающие и прыгающие элементы.

Запрещены надписи вида «осталось N мест».

## Фото и соцсети

На экранах доверия и в пейволле использовать настоящие фотографии Кати Усмановой как социальное доказательство.

Разрешённые источники: сайт проекта и аккаунты Кати.

В квизе нельзя размещать логотипы и ссылки на Instagram или Meta.

## ГетКурс

Свою форму оформления заказа не делать.

На пейволле нужно встроить эмбед формы создания заказа из ГетКурса.

Ссылку на эмбед должен дать заказчик.

Если эмбед ещё не выдан, поставить временный placeholder и явно пометить TODO.
