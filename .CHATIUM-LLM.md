# Квиз-воронка «Усманова Фит»

Многошаговый квиз для подписки на фитнес-программы Екатерины Усмановой: вопросы → расчёт персонального результата (КБЖУ/ИМТ) → пейволл с тарифами и формой GetCourse.

## Архитектура

- `index.tsx` — серверная обёртка маршрута (`indexRoute`, GET `/`). Рендерит `<head>` со стилями квиза (`quizThemeCss`), монтирует `QuizPage.vue`, пробрасывает UTM в `window.__QUIZ_BOOT__` и держит скрытый контейнер `#gcPaywallForm` для формы GetCourse.
- `pages/QuizPage.vue` — корневой Vue-компонент: состояние прохождения, навигация, расчёт результата, сохранение лида, выбор экрана по `screen.kind`.
- `components/*.vue` — экраны и элементы: `QuestionScreen`, `TrustScreen`, `LoaderScreen`, `LeadGateScreen`, `ResultScreen`, `PaywallScreen`, плюс `QuizHeader`, `Button`, `OptionButton`, `GetCourseForm`, иконки.
- `src/quiz/*` — изоморфная (`// @shared`) бизнес-логика: `types`, `screens` (нормализация карты экранов), `state` (валидация/навигация), `result` (BMR/КБЖУ/ИМТ по Миффлину-Сан Жеору), `programs` (подбор программ), `resultImages`.
- `src/ui/*` — дизайн-токены (`tokens.ts`) и собранный CSS (`theme.ts`).
- `data/quizScreenMap.ts` — `// @shared` модуль с конфигурацией всех экранов, тарифов и юридических ссылок (вместо JSON, т.к. Chatium не поддерживает импорт `.json`).
- `shared/*` — `utm.ts` (UTM из URL), `quizPersist.ts` (localStorage прогресс/сессия), `saveLead.ts` (отправка лида).

## Данные

- `tables/leads.table` (`quiz_leads`) — лиды квиза: sessionId, name, email, consentAccepted, answers, рост/вес, resultType, calorieNorm, macros, utm, status (`draft`/`completed`).
- `api/leads/upsert.ts` (`leadsUpsertRoute`, POST) — идемпотентный upsert лида по `sessionId`.

## Ключевые решения для Chatium

- Импорт данных только через `.ts`-модуль, не `.json`.
- Все frontend-импортируемые модули логики помечены `// @shared`.
- `shared/saveLead.ts` отправляет лид по `leadsUpsertRoute.url()` (без хардкода URL).
- Серверный JSX использует `charset` (не `charSet`).
